# Beginner JavaScript Questions

### What is JavaScript, and what are its main uses?

**Interview-style answer:**
JavaScript is a high-level, interpreted, and dynamically typed programming language primarily used to add interactivity and dynamic behavior to web pages. Beyond browsers, it’s also used on servers with Node.js, in mobile applications, and in desktop development.

**Explanation:**

- Runs inside web browsers as the engine for client-side logic.
- Used to manipulate the DOM, handle events, validate user input, perform AJAX calls.
- With Node.js, it powers backend servers.
- Widely used in frameworks (React, Angular, Vue) for building full-stack apps.

---

### What are the differences between var, let, and const?

**Interview-style answer:**

- `var` is function-scoped and allows redeclaration.
- `let` is block-scoped and does not allow redeclaration.
- `const` is block-scoped and cannot be reassigned once declared.

**Explanation:**

- `var`: Hoisted with undefined, can lead to bugs due to scope leakage.
- `let`: Safer; exists only inside `{}` block.
- `const`: Same as `let` in scope, but assignment is fixed (though objects/arrays can still be mutated).

_**Function scope**_ - A variable is accessible everywhere inside the function it’s declared in.

_**Block scope**_ - A variable can be accessed only within the curly braces `{}` where it was declared, such as in an `if`, `for`, or `while` block.

### What are primitive data types in JavaScript?

**Interview-style answer:**
JavaScript has 7 primitive types: string, number, boolean, null, undefined, symbol, and bigint. These are immutable and compared by value.

**Explanation:**

- Primitive types don’t have methods of their own (though wrapper objects give temporary support).
- Example: numbers (`42`), booleans (`true`), `undefined`, `null`, `Symbol('id')`, and `BigInt(123n)`.

### What is the difference between undefined and null in JavaScript?

**Interview-style answer:**
`undefined` means a variable has been declared but not assigned a value, while `null` is an intentional assignment representing “no value”.

**Explanation:**

- `let a; console.log(a); // undefined`
- `let b = null;` means developer explicitly set it to empty.

### What are the differences between == and === operators in JavaScript?

**Interview-style answer:**

- `==` is loose equality: it compares values after type coercion.
- `===` is strict equality: it checks both value and type with no coercion.

**Explanation:**

- `5 == "5"` → true, but `5 === "5"` → false.
- Always prefer `===` to avoid unexpected results.

---

### Explain the concept of type coercion in JavaScript.

**Interview-style answer:**
Type coercion is the automatic or implicit conversion of values from one data type to another during comparisons or operations in JavaScript.

**Explanation:**

- `"5" + 10` → `"510"` (string coercion).
- `5 - "2"` → 3 (number coercion).
- To avoid confusion, use strict equality and explicit conversions.

### What are JavaScript functions and how do you declare them?

**Interview-style answer:**
Functions are reusable blocks of code that can be defined using function declarations, function expressions, or arrow functions.

**Explanation:**

- Declared:

```js
function greet(name) {
  return "Hello " + name;
}
```

- Expression:

```js

  return "Hello " + name;
};
```

- Arrow:

```js
const greet = (name) => "Hello " + name;
```

### What is hoisting in JavaScript?

**Interview-style answer:**
Hoisting is JavaScript’s behavior of moving variable and function declarations to the top of their scope before execution.

**Explanation:**

- `var` declarations are hoisted with value `undefined`.
- Functions are hoisted fully (you can call them before declaration).
- `let` and `const` are hoisted but in a _temporal dead zone_ until assigned.

---

### What is the Temporal Dead Zone?

**Interview-style answer:**
The temporal dead zone is the time between entering a scope (like a block) and the actual line where `let` or `const` is declared. You cannot access the variable in this period; accessing it throws a `ReferenceError`.

### What is the difference between synchronous and asynchronous code in JavaScript?

**Interview-style answer:**
Synchronous code runs line by line, blocking further execution until complete. Asynchronous code allows other operations to continue by using callbacks, promises, or async/await.

**Explanation:**

- Sync: Blocks program. Example: reading a file with `fs.readFileSync`.
- Async: Non-blocking. Example: `setTimeout`, AJAX requests, database queries.

### What is the purpose of bind(), call(), and apply() methods in JavaScript?

**Interview-style answer:**
All three explicitly set the `this` context for a function.

- `call` runs the function immediately with arguments individually.
- `apply` runs the function immediately with arguments as an array.
- `bind` returns a new function with `this` fixed, without executing immediately.

**Explanation:**

```js
function greet(greeting) {
  console.log(greeting + ", " + this.name);
}
const person = { name: "Sam" };

greet.call(person, "Hello");
greet.apply(person, ["Hi"]);
const boundFunc = greet.bind(person, "Hey");
boundFunc();
```

### How does the `this` keyword work in JavaScript?

**Interview-style answer:**
`this` refers to the current execution context, and its value depends on how a function is called, not where it is written.

- Global context → window/global object.
- Inside an object method → the object.
- Inside a function → undefined (in strict mode).
- Arrow functions → lexical `this` from surrounding scope.

Arrow functions do not have their own `this`; they inherit it from surrounding scope, which is often the global context.


### What is the difference between filter() and find() methods in JavaScript arrays?

**Interview-style answer:**

- `filter()` returns a new array with all elements that pass a test function.
- `find()` returns the first element that passes a test function, or undefined if none found.

**Explanation:**

```js
const numbers = [1, 2, 3, 4, 5];

// filter() - returns array of all matches
const evens = numbers.filter((n) => n % 2 === 0); // [2, 4]

// find() - returns first match only
const firstEven = numbers.find((n) => n % 2 === 0); // 2
```

### What is the difference between loops - for, for...In and for...of methods in JavaScript arrays?

**Interview-style answer:**

- `for` loop: Traditional loop with index control
- `for...in` loop: Iterates over enumerable properties (including inherited ones)
- `for...of` loop: Iterates over iterable values (arrays, strings, etc.)

**Explanation:**

```js
const arr = ["a", "b", "c"];

// for loop
for (let i = 0; i < arr.length; i++) {
  console.log(arr[i]); // 'a', 'b', 'c'
}

// for...in (not recommended for arrays)
for (let index in arr) {
  console.log(arr[index]); // 'a', 'b', 'c'
}

// for...of (recommended for arrays)
for (let value of arr) {
  console.log(value); // 'a', 'b', 'c'
}
```

### Explain reduce(), map(), filter() methods in JavaScript arrays?

**Interview-style answer:**

- `map()`: Transforms each element and returns new array
- `filter()`: Creates new array with elements that pass a test
- `reduce()`: Reduces array to single value using accumulator

**Explanation:**

```js
const numbers = [1, 2, 3, 4, 5];

```

### What is debouncing and throttling in JavaScript?

**Interview-style answer:**

- **Debouncing**: Delays function execution until after events stop firing for a specified time
- **Throttling**: Limits function execution to once per specified time period

**Explanation:**

```js
// Debouncing - waits for pause in events
function debounce(func, delay) {
  let timeoutId;
  return function (...args) {
    clearTimeout(timeoutId);
    timeoutId = setTimeout(() => func.apply(this, args), delay);
  };
}

// Throttling - limits execution frequency
function throttle(func, limit) {
  let inThrottle;
  return function (...args) {
    if (!inThrottle) {
      func.apply(this, args);
      inThrottle = true;
      setTimeout(() => (inThrottle = false), limit);
    }
  };
}
```

### How would you implement a deep clone of an object without using libraries?

**Interview-style answer:**
Use recursive approach with proper handling of different data types, circular references, and special objects.

**Explanation:**

```js
function deepClone(obj, visited = new WeakMap()) {
  // Handle null/undefined
  if (obj === null || typeof obj !== "object") return obj;

  // Handle circular references
  if (visited.has(obj)) return visited.get(obj);

  // Handle Date
  if (obj instanceof Date) return new Date(obj.getTime());

  // Handle Array
  if (Array.isArray(obj)) {
    const cloned = [];
    visited.set(obj, cloned);
    obj.forEach((item) => cloned.push(deepClone(item, visited)));
    return cloned;
  }

  // Handle Object
  const cloned = {};
  visited.set(obj, cloned);
  Object.keys(obj).forEach((key) => {
    cloned[key] = deepClone(obj[key], visited);
  });

  return cloned;
}
```

### What is the event loop in JavaScript, and how does it work?

**Interview-style answer:**
The event loop is JavaScript's mechanism for handling asynchronous operations. It continuously checks the call stack and task queue, moving tasks from the queue to the stack when the stack is empty.

**Explanation:**

```js
console.log("1"); // Synchronous - goes to call stack immediately

setTimeout(() => console.log("2"), 0); // Asynchronous - goes to Web APIs

Promise.resolve().then(() => console.log("3")); // Microtask - higher priority

console.log("4"); // Synchronous

// Output: 1, 4, 3, 2
```

The event loop has:

- **Call Stack**: Executes synchronous code
- **Web APIs**: Handle async operations (setTimeout, DOM events)
- **Task Queue**: Callback functions waiting to execute
- **Microtask Queue**: Higher priority than task queue (Promises)

### How does prototypal inheritance work in JavaScript?

**Interview-style answer:**
JavaScript uses prototypal inheritance where objects inherit properties and methods from their prototype chain. Every object has a `__proto__` property pointing to its prototype.

**Explanation:**

```js
// Constructor function
function Person(name) {
  this.name = name;
}

// Add method to prototype
Person.prototype.greet = function () {
  return `Hello, I'm ${this.name}`;
};

const person = new Person("John");
console.log(person.greet()); // "Hello, I'm John"

// Inheritance
function Student(name, grade) {
  Person.call(this, name);
  this.grade = grade;
}

Student.prototype = Object.create(Person.prototype);
Student.prototype.constructor = Student;

const student = new Student("Jane", "A");
console.log(student.greet()); // Inherits from Person
```

**Explanation:**


### How does Object.create() work in JavaScript?

**Interview-style answer:**
`Object.create()` creates a new object with the specified prototype object and properties. It's an alternative to constructor functions for creating objects with inheritance.

**Explanation:**

```js
const personPrototype = {
  greet() {
    return `Hello, I'm ${this.name}`;
  },

const person = Object.create(personPrototype);
person.name = "John";

console.log(person.greet()); // "Hello, I'm John"
console.log(person.__proto__ === personPrototype); // true

// Creating object with properties
const student = Object.create(personPrototype, {
  name: { value: "Jane", writable: true },

});
```

### What are the advantages of using const over let or var?

**Interview-style answer:**
`const` provides immutability, prevents accidental reassignment, makes code more predictable, and helps catch bugs early. It's block-scoped like `let` but cannot be reassigned.

**Explanation:**

```js
// const advantages
const PI = 3.14159; // Cannot be reassigned
const config = { api: "https://api.example.com" };
// config = {}; // Error: Assignment to constant variable

// But object properties can still be modified
config.api = "https://new-api.com"; // This works

// Use const for arrays too
const numbers = [1, 2, 3];
numbers.push(4); // This works
// numbers = []; // Error: Cannot reassign

// Benefits:
// 1. Prevents accidental reassignment
// 2. Makes intent clear (this value shouldn't change)
// 3. Better for optimization by JavaScript engine
```

### Explain the concept of async/await in JavaScript.

**Interview-style answer:**
`async/await` is syntactic sugar over Promises that makes asynchronous code look and behave more like synchronous code. It uses `async` functions and `await` keyword to pause execution until Promise resolves.

**Explanation:**

```js

    .then((response) => response.json())
    .then((user) => {

    })
    .then((response) => response.json())
    .catch((error) => console.error("Error:", error));

async function fetchUserData() {
  try {
    const userResponse = await fetch("/api/user");
    const user = await userResponse.json();

    const postsResponse = await fetch(`/api/posts/${user.id}`);
    const posts = await postsResponse.json();

    return posts;
  } catch (error) {
    console.error("Error:", error);
  }
}

// Benefits:
// 1. More readable than Promise chains
// 2. Easier error handling with try/catch
// 3. Debugging is simpler
// 4. Looks like synchronous code
```

### What is event bubbling and event capturing in JavaScript?

**Interview-style answer:**
Event bubbling propagates events from the target element up to the root, while event capturing propagates from root down to the target. Both phases occur unless stopped.

**Explanation:**

```js

//   <div id="parent">
//     <div id="child">Click me</div>

const child = document.getElementById("child");

  "click",
  () => console.log("Grandparent captured"),
  true
);

grandparent.addEventListener(
  "click",

);

child.addEventListener("click", () => console.log("Child bubbled"), false);

// 1. Grandparent captured
// 2. Parent captured
// 3. Child captured
// 4. Child bubbled
// 5. Parent bubbled
// 6. Grandparent bubbled

child.addEventListener("click", (e) => {
  e.stopPropagation(); // Stops further propagation
  console.log("Event stopped");
```

### How does the new keyword work in JavaScript?

**Interview-style answer:**
The `new` keyword creates a new object, sets its prototype to the constructor's prototype, binds `this` to the new object, and returns the object (unless constructor returns a different object).

```js
  return typeof result === "object" ? result : obj;
```

### What is the importance of the bind() method in JavaScript?

### What is memoization in JavaScript and how can it be implemented?

**Interview-style answer:**
Memoization is an optimization technique that caches function results to avoid redundant calculations. It trades memory for speed by storing results of expensive function calls.

**Explanation:**

```js
// Basic memoization
function memoize(fn) {
  const cache = new Map();

  return function (...args) {
    const key = JSON.stringify(args);

    if (cache.has(key)) {
      console.log("Cache hit!");
      return cache.get(key);
    }

    console.log("Computing...");
    const result = fn.apply(this, args);
    cache.set(key, result);
    return result;
  };
}

// Example: Fibonacci
function fibonacci(n) {
  if (n <= 1) return n;
  return fibonacci(n - 1) + fibonacci(n - 2);
}

const memoizedFibonacci = memoize(fibonacci);

console.log(memoizedFibonacci(10)); // Computing... (slow)
console.log(memoizedFibonacci(10)); // Cache hit! (fast)

// Advanced memoization with cache size limit
function memoizeWithLimit(fn, limit = 100) {
  const cache = new Map();

  return function (...args) {
    const key = JSON.stringify(args);

    if (cache.has(key)) {
      // Move to end (LRU)
      const value = cache.get(key);
      cache.delete(key);
      cache.set(key, value);
      return value;
    }

    const result = fn.apply(this, args);

    // Remove oldest if at limit
    if (cache.size >= limit) {
      const firstKey = cache.keys().next().value;
      cache.delete(firstKey);
    }

    cache.set(key, result);
    return result;
  };
}

// React useMemo equivalent
function useMemo(fn, deps) {
  const [memoizedValue, setMemoizedValue] = useState(() => fn());
  const [prevDeps, setPrevDeps] = useState(deps);

  useEffect(() => {
    if (JSON.stringify(deps) !== JSON.stringify(prevDeps)) {
      setMemoizedValue(fn());
      setPrevDeps(deps);
    }
  }, deps);

  return memoizedValue;
}
```

### Explain the concept of currying in JavaScript.

**Interview-style answer:**
Currying transforms a function that takes multiple arguments into a sequence of functions that each take a single argument. It enables partial application and functional composition.

**Explanation:**

```js

  return a + b + c;
}

// Curried version
function curriedAdd(a) {
  return function (b) {
    return function (c) {
      return a + b + c;
    };
  };
}

// Arrow function version
const curriedAddArrow = (a) => (b) => (c) => a + b + c;

// Usage
console.log(add(1, 2, 3)); // 6
console.log(curriedAdd(1)(2)(3)); // 6
console.log(curriedAddArrow(1)(2)(3)); // 6

// Partial application
const addOne = curriedAdd(1);
const addOneAndTwo = addOne(2);
console.log(addOneAndTwo(3)); // 6

// Generic curry function
function curry(fn) {
  return function curried(...args) {
    if (args.length >= fn.length) {
      return fn.apply(this, args);
    } else {
      return function (...nextArgs) {
        return curried.apply(this, args.concat(nextArgs));
      };
    }
  };
}

// Example usage

const curriedMultiply = curry(multiply);

console.log(curriedMultiply(2, 3)(4)); // 24
console.log(curriedMultiply(2)(3, 4)); // 24

// Real-world example: Event handling
const addEventListener = curry((event, handler, element) => {
  element.addEventListener(event, handler);
});

const addClickToButton = addClickHandler(() => console.log("Clicked!"));
```

### How do you implement deep cloning of objects in JavaScript?

**Interview-style answer:**
Deep cloning creates a complete copy of an object including all nested objects and arrays. It can be implemented recursively or using JSON methods (with limitations).

**Explanation:**

```js
function deepClone(obj, visited = new WeakMap()) {
  if (obj === null || typeof obj !== "object") return obj;
  if (visited.has(obj)) return visited.get(obj);

  let cloned;

  if (obj instanceof Date) {
    cloned = new Date(obj.getTime());
  } else if (obj instanceof Array) {
    cloned = [];
    visited.set(obj, cloned);
    obj.forEach((item, index) => {
      cloned[index] = deepClone(item, visited);
    });
  } else if (obj instanceof RegExp) {
    cloned = new RegExp(obj.source, obj.flags);
  } else {
    cloned = {};
    visited.set(obj, cloned);
    Object.keys(obj).forEach((key) => {
      cloned[key] = deepClone(obj[key], visited);
    });
  }

  return cloned;
}
```

### How do you implement a custom JavaScript engine or interpreter?

**Interview-style answer:**
A JavaScript engine consists of a parser (creates AST), compiler (generates bytecode), and runtime (executes code). Key components include lexical analysis, syntax parsing, scope resolution, and execution context management.

**Explanation:**

```js
// Simplified JavaScript interpreter
class SimpleInterpreter {
  constructor() {
    this.globalScope = new Map();
    this.scopeStack = [this.globalScope];
  }

  // Lexical analysis
  tokenize(code) {
    const tokens = [];
    const regex = /\s*(\d+|\w+|[+\-*/=(){};])\s*/g;
    let match;

    while ((match = regex.exec(code)) !== null) {
      tokens.push(match[1]);
    }
    return tokens;
  }

  // Simple parser
  parse(tokens) {
    let index = 0;

    function parseExpression() {
      if (tokens[index] === "(") {
        index++; // consume '('
        const expr = parseExpression();
        index++; // consume ')'
        return expr;
      }

      const token = tokens[index++];
      if (/^\d+$/.test(token)) {
        return { type: "number", value: parseInt(token) };
      }

      if (/^\w+$/.test(token)) {
        return { type: "variable", name: token };
      }

      throw new Error(`Unexpected token: ${token}`);
    }

    return parseExpression();
  }

  // Execution
  execute(ast) {
    switch (ast.type) {
      case "number":
        return ast.value;
      case "variable":
        return this.getVariable(ast.name);
      default:
        throw new Error(`Unknown AST type: ${ast.type}`);
    }
  }

  getVariable(name) {
    for (let i = this.scopeStack.length - 1; i >= 0; i--) {
      if (this.scopeStack[i].has(name)) {
        return this.scopeStack[i].get(name);
      }
    }
    throw new Error(`Variable ${name} not found`);
  }
}

// Usage
const interpreter = new SimpleInterpreter();
const tokens = interpreter.tokenize("(42)");
const ast = interpreter.parse(tokens);
const result = interpreter.execute(ast);
console.log(result); // 42
```

```js
    if (!this.events[event]) this.events[event] = [];
    this.events[event].push(callback);
  }

  emit(event, data) {
    if (this.events[event]) {
      this.events[event].forEach((callback) => callback(data));
    }
  }
}

// Factory Pattern
class ShapeFactory {
  static createShape(type, ...args) {

        return new Circle(...args);
      case "rectangle":
        return new Rectangle(...args);
      default:
        throw new Error("Unknown shape type");
    }
  }
}
```

### Explain the concept of JavaScript profiling and performance optimization.

**Interview-style answer:**
JavaScript profiling involves measuring code execution time, memory usage, and identifying bottlenecks. Tools like Chrome DevTools, Node.js profiler, and performance APIs help optimize applications.

**Explanation:**

```js
// Performance API
const startTime = performance.now();
// Expensive operation
const endTime = performance.now();
console.log(`Operation took ${endTime - startTime} milliseconds`);

// Memory profiling
const memBefore = performance.memory;
// Memory-intensive operation
const memAfter = performance.memory;
console.log(
  `Memory used: ${memAfter.usedJSHeapSize - memBefore.usedJSHeapSize}`
);

// Custom profiler
class Profiler {
  constructor() {
    this.marks = new Map();
  }

  start(label) {
    this.marks.set(label, performance.now());
  }

  end(label) {
    const start = this.marks.get(label);
    const duration = performance.now() - start;
    console.log(`${label}: ${duration}ms`);
    return duration;
  }
}
```
