### What is React? How does it differ from other JavaScript frameworks?

**Interview-style answer:**
React is a JavaScript library for building user interfaces, particularly web applications. It focuses on component-based architecture, virtual DOM for
performance, and unidirectional data flow. Unlike frameworks like Angular, React is just the view layer.

**Explanation:**

```jsx
// React component example
import React, { useState } from "react";

function Welcome({ name }) {
  const [count, setCount] = useState(0);

  return (
    <div>
      <h1>Hello, {name}!</h1>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

// Key differences from other frameworks:
// 1. Component-based: Everything is a component
// 2. Virtual DOM: Efficient diffing and updates
// 3. JSX: HTML-like syntax in JavaScript
// 4. Unidirectional data flow: Props down, events up
// 5. Library vs Framework: React is focused on UI, not full solution
```

### Explain the concept of a component in React.

**Interview-style answer:**
Components are reusable, self-contained pieces of UI that encapsulate structure, behavior, and styling. They accept props as input and return JSX
describing what should render.

**Explanation:**

```jsx
// Functional Component
function UserCard({ user, onEdit }) {
  return (
    <div className="user-card">
      <img src={user.avatar} alt={user.name} />
      <h3>{user.name}</h3>
      <p>{user.email}</p>
      <button onClick={() => onEdit(user.id)}>Edit</button>
    </div>
  );
}

// Class Component (legacy)
class UserCard extends React.Component {
  render() {
    const { user, onEdit } = this.props;
    return (
      <div className="user-card">
        <img src={user.avatar} alt={user.name} />
        <h3>{user.name}</h3>
        <p>{user.email}</p>
        <button onClick={() => onEdit(user.id)}>Edit</button>
      </div>
    );
  }
}

// Component composition
function UserList({ users }) {
  const handleEdit = (userId) => {
    console.log("Editing user:", userId);
  };

  return (
    <div className="user-list">
      {users.map((user) => (
        <UserCard key={user.id} user={user} onEdit={handleEdit} />
      ))}
    </div>
  );
}
```

### What are React hooks? Can you name a few common ones?

**Interview-style answer:**
Hooks are functions that let you use state and lifecycle features in functional components. Common hooks include useState, useEffect, useContext,
useReducer, useMemo, and useCallback.

**Explanation:**

```jsx
import React, { useState, useEffect, useMemo, useCallback } from "react";

function ExampleComponent() {
  // useState - manage component state
  const [count, setCount] = useState(0);
  const [user, setUser] = useState(null);

  // useEffect - side effects (lifecycle replacement)
  useEffect(() => {
    document.title = `Count: ${count}`;

    return () => {
      document.title = "React App";
    };
  }, [count]);

  // useMemo - expensive calculations
  const expensiveValue = useMemo(() => {
    return count * 1000;
  }, [count]);

  // useCallback - memoize functions
  const handleIncrement = useCallback(() => {
    setCount((prev) => prev + 1);
  }, []);

  return (
    <div>
      <p>Count: {count}</p>
      <p>Expensive Value: {expensiveValue}</p>
      <button onClick={handleIncrement}>Increment</button>
    </div>
  );
}
```

### What is the difference between controlled and uncontrolled components?

**Interview-style answer:**
Controlled components have their form data handled by React state - the component controls the input value. Uncontrolled components store their data in
the DOM itself, accessed via refs.

**Explanation:**

```jsx
import React, { useState, useRef } from "react";

// Controlled Component
function ControlledForm() {
  const [name, setName] = useState("");

  return (
    <form>
      <input
        type="text"
        value={name}
        onChange={(e) => setName(e.target.value)}
        placeholder="Name"
      />
    </form>
  );
}

// Uncontrolled Component
function UncontrolledForm() {
  const nameRef = useRef();

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(nameRef.current.value);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input ref={nameRef} type="text" defaultValue="" placeholder="Name" />
    </form>
  );
}
```

### What is the useEffect hook and how does it work?

**Interview-style answer:**
useEffect lets you perform side effects in functional components. It combines componentDidMount, componentDidUpdate, and componentWillUnmount lifecycle
methods. It runs after every render by default.

**Explanation:**

```jsx
import React, { useState, useEffect } from "react";

function UserProfile({ userId }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  // Effect with dependency array
  useEffect(() => {
    setLoading(true);

    fetch(`/api/users/${userId}`)
      .then((response) => response.json())
      .then((userData) => {
        setUser(userData);
        setLoading(false);
      });
  }, [userId]); // Only re-run when userId changes

  // Effect with cleanup
  useEffect(() => {
    const timer = setInterval(() => {
      console.log("Timer tick");
    }, 1000);

    // Cleanup function (componentWillUnmount)
    return () => {
      clearInterval(timer);
    };
  }, []); // Empty array = run once on mount

  if (loading) return <div>Loading...</div>;

  return (
    <div>
      <h2>{user?.name}</h2>
      <p>{user?.email}</p>
    </div>
  );
}
```

### What is React Context and how do you use it?

**Interview-style answer:**
React Context provides a way to pass data through the component tree without having to pass props down manually at every level. It's useful for global
state like themes, authentication, or language preferences.

**Explanation:**

```jsx
import React, { createContext, useContext, useReducer } from "react";

// Create context
const AuthContext = createContext();

// Context provider
function AuthProvider({ children }) {
  const [state, dispatch] = useReducer(authReducer, {
    user: null,
    isAuthenticated: false,
    loading: false,
  });

  const login = async (email, password) => {
    dispatch({ type: "LOGIN_START" });
    try {
      const user = await authService.login(email, password);
      dispatch({ type: "LOGIN_SUCCESS", payload: user });
    } catch (error) {
      dispatch({ type: "LOGIN_ERROR", payload: error.message });
    }
  };

  const logout = () => {
    authService.logout();
    dispatch({ type: "LOGOUT" });
  };

  return (
    <AuthContext.Provider value={{ ...state, login, logout }}>
      {children}
    </AuthContext.Provider>
  );
}

// Custom hook to use context
function useAuth() {
  const context = useContext(AuthContext);
  if (!context) {
    throw new Error("useAuth must be used within AuthProvider");
  }
  return context;
}

// Component using context
function LoginButton() {
  const { isAuthenticated, login, logout } = useAuth();

  if (isAuthenticated) {
    return <button onClick={logout}>Logout</button>;
  }

  return (
    <button onClick={() => login("user@example.com", "password")}>Login</button>
  );
}

// Reducer
function authReducer(state, action) {
  switch (action.type) {
    case "LOGIN_START":
      return { ...state, loading: true };
    case "LOGIN_SUCCESS":
      return {
        ...state,
        user: action.payload,
        isAuthenticated: true,
        loading: false,
      };
    case "LOGIN_ERROR":
      return { ...state, loading: false, error: action.payload };
    case "LOGOUT":
      return { user: null, isAuthenticated: false, loading: false };
    default:
      return state;
  }
}
```

Hooks are functions that let you use state and lifecycle features in functional components. Common hooks include useState, useEffect, useContext,
useReducer, useMemo, and useCallback.

```jsx
  const expensiveValue = useMemo(() => {

  const handleIncrement = useCallback(() => {

      <p>Expensive Value: {expensiveValue}</p>

// Custom hook
function useCounter(initialValue = 0) {
  const [count, setCount] = useState(initialValue);

  const increment = () => setCount((c) => c + 1);
  const decrement = () => setCount((c) => c - 1);
  const reset = () => setCount(initialValue);

  return { count, increment, decrement, reset };
}
```

### What is JSX? How does React use JSX to render components?

**Interview-style answer:**
JSX is a syntax extension for JavaScript that allows you to write HTML-like markup inside JavaScript. React transforms JSX into React.createElement()
calls, which create virtual DOM elements.

**Explanation:**

```jsx
// JSX syntax
const element = <h1>Hello, World!</h1>;

// Compiles to:
const element = React.createElement("h1", null, "Hello, World!");

// JSX with props and children
const greeting = (
  <div className="greeting">
    <h1>Welcome!</h1>
    <p>Nice to see you here.</p>
  </div>
);

// Compiles to:
const greeting = React.createElement(
  "div",
  { className: "greeting" },
  React.createElement("h1", null, "Welcome!"),
  React.createElement("p", null, "Nice to see you here.")
);

// JSX with expressions
const user = { name: "John", age: 30 };
const userProfile = (
  <div>
    <h2>{user.name}</h2>
    <p>Age: {user.age}</p>
    <p>Is Adult: {user.age >= 18 ? "Yes" : "No"}</p>
  </div>
);

// JSX rules:
// 1. Must return single element (or Fragment)
// 2. Self-closing tags must end with />
// 3. Use camelCase for attributes (className, onClick)
// 4. Use {} for JavaScript expressions
```

### Explain the virtual DOM in React and how it optimizes performance.

The Virtual DOM is a JavaScript representation of the real DOM kept in memory. React compares (diffs) the current and previous virtual DOM trees, then
updates only the changed parts in the real DOM, minimizing expensive DOM operations.
